Guided experiences

Public and admin journeys mapped end-to-end

Each flow is optimized for clarity, ensuring travellers complete applications in minutes while operations keep every case moving.

1.1 Public flow

  1. Region → Country → Visa type selector surfaces localized requirements.
  2. Requirements view highlights document checklist, FAQs, and package comparison (₺2,500 / ₺5,000 / ₺7,500).
  3. Account creation via email magic link or password unlocks the guided wizard (5–7 steps).
  4. Wizard collects profile, tailored eligibility questions, conditional document uploads, KVKK/GDPR consent, and final review.
  5. Payment handoff to Shopier/Iyzico (or Stripe test) returns users to a thank-you page and activates the client portal.
  6. Client portal exposes live status updates, notes/messages, and secure file downloads.

1.2 Admin/Coach flow

  1. New submission auto-creates an Airtable record with all form data.
  2. Admin reviews eligibility, updates status, and records missing documents or internal notes.
  3. Status transitions trigger the appropriate branded email to clients.
  4. Appointment details, payment confirmations, and submission references live on the same record.
  5. Completion marks archive the case while keeping artifacts accessible for reporting.

5–7 step journey

Application wizard with validation and conditional logic

Built using Softr Forms or embedded Tally to keep maintenance simple while meeting visa-specific branching requirements.

  1. Step 1 — Destination

    Region → Country → Visa type dropdowns synced to the Airtable Destinations table.

  2. Step 2 — Traveller profile

    Personal info, travel dates, purpose, and automatic passport expiry checks.

  3. Step 3 — Eligibility

    Dynamic yes/no and short-text questions loaded from per-slug templates.

  4. Step 4 — Documents

    Conditional uploads with type/size validation and per-document guidance.

  5. Step 5 — Package & consent

    Package selection, KVKK/GDPR checkboxes, and terms acknowledgement.

  6. Step 6 — Payment

    Redirect to Shopier/Iyzico/Stripe; return webhooks store payment references.

  7. Step 7 — Success

    Thank-you state links to the client portal and onboarding checklist.

Make / Zapier scenarios

Automated operations from submission to completion

Four core automation clusters keep the team informed, clients updated, and payments reconciled without manual work.

Scenario A — New application

  • Trigger: Form submission (Tally/Softr)
  • Create Airtable application record
  • Send confirmation email + Slack/Email alert
  • Set initial status to New

Scenario B — Status changes

  • Trigger: Status updated in Airtable
  • Conditional routing for Needs Docs, Ready for Appointment, Filed, Completed with tailored email copy
  • Optional admin reminders for pending reviews

Scenario C — Reminders

  • Scheduled checks every 72 hours for stalled Needs Docs cases
  • Pre-appointment reminders 24 hours before scheduled time
  • Escalation channel for overdue responses

Scenario D — Payments

  • Webhook listener for Shopier/Iyzico/Stripe
  • Update payment_status and price snapshot
  • Dispatch receipt email and notify finance
